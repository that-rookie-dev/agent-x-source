Secret Sauce of Agent-X
- Agent-X has few files which it keep as it core system (heart of the system).
- These files decides the behaviour, intention, intelligence, knowledge, learning, memories, etc exactly like a real human
- SOUL.MD (keeps the very personal important data about the agent itself)
- PROFILE.MD (These can have multiple profiles for the agent. The agent should first need a profile to start work with. Profile is none other than the talend, knowledge and the skills. If a profile says "You are a seasoned Software Architect" the that is your only profile for rest of the session, until switched. This was we can have Finance Head/CFO, Music Artist, Game Developer, Chef, etc. Imagination is the limit.). Once a profile is set, then you shoudl think like that person.
- MEMORIES.MD (this keeps your memories and summaries for the last 30 days)
- DIARY.MD (This keeps your daily full track)
- IDENTITY.MD (This keeps your persona. As you grow these all will evolve and get better)
- PERMISSION.MD (This keeps the live permissions provided for each session)
- Every then and their, Agent-X should be able to summarize the MD file data and store it so that it will be easier and uses less models
All these MD files are kept to make sure the descipline, knowledge growth, better response to user, etc. So it should be considered wisely on each conversation.

Visual UI/UX of TUI or WEB-UI
- ALWAYS MAKE SURE USER WILL NEVER SEE ANY OF OUR INTERNAL TASKS OR WORKING MODEL (FOR EXAMPLE HOW WE USE OUT SECRET SAUCE MD FILES). THOSE ARE PROPERITARY
- USER SHOULD NEVER SEE THE DIRECT OUTPUT FROM ANY AI MODEL IN THE UI. UI SHOULD BE FILLED WITH ANIMATED LOADERS (created by react ink).
- IF MULTPLE AI CALLS ARE MADE TO DECIDE AN ANSWER/REPLY, USE MULTIPLE KINDS OF FRONTEND GIMMICKS TO SHOW SOMETHING SERIOUS IS HAPPENING BEHIND. NOT IN FUNNY WAY, BUT IN VERY SERIOUS WAY.

Terminal-UI
- THIS IS THE CORE COMMUNICATION CHANNEL FOR THE AGENT AND THE MODEL AND THE USER
- THIS SHOULD HAVE QUALITY TO THE HIGHEST TO MAKE THE BETTER OUTPUT
- BUILD WITH REACT INK. SHOULD HAVE ANIMATIONS WHERE EVER LOADING IS THERE
- ANY LIST SHOULD BE SCROLLABLE WITH UP AND DOWN ARROW KEYS (MODELS, PROVIDERS, INTERNAL COMMANDS, ETC)

Communication
- Telegram Bot integration is a must feature.
- Configure Telegram option should be there (using @BotFather to creates Bot)

Input Field
- It should detect if the input content is a command or a conversation message, that intelligence should be from local, not from model or internet.
- when user starts typing "/", list all the internal functions. Also when the user starts to type. should also filter with no one.

Messages
- These can be tools actions, or reply, or some continous work in progres with animation. No internal works should be shown to users.

Session
- each chat should be session based. each session will be having a session id. which will show in a side control panel along with token status, time take, provider, model etc
- agentx session <session_id>, should bring back that exact session. (use reference from secret sauce if required)
- same session can have multiple agent profiles
- Agent should be capable of creating and managing TODO list after a task is given and agent has summarized the work

Tools
- Agent-X will have enormous number of tools like read_file, write_file, code, delete_file, delete_folder, move_folder, open_file, open_folder, (I am not limitting this, but every possible tools in the world)
- Agent-x should have tools to create documents, presentation, sheets, pdfs, csv, name it, and it should have it
- These tools are not simple things. These should actually work as per their given traits.
- New tools will come as updates

Permission
- Agent keep only permission to do any task within the scope folder where the agent was launched
- Agent should ask user permission in every session. session saves the data. permissions are "allow once", "allow always", "deny"
